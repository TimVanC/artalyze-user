BETA Version 1.4 Patch Notes

Release Date: February 27th, 2025

New Features & Improvements

Pinch-to-Zoom for Enlarged Images

  • Users on mobile devices (width < 650px) can now pinch-to-zoom when viewing an enlarged image.
  • Zooming is only enabled for enlarged images, preventing unintended page zooming.
  • Users can zoom in and reset to the original size, but cannot zoom out beyond 100% to avoid layout issues.

Enhanced Mobile Zoom Controls

  • Prevented zooming out past the original image size to avoid excessive white space on Chrome.
  • Enforced a minimum zoom level of 100% while allowing smooth zooming in.
  • Improved image scaling to ensure zoom remains intuitive across different devices.

Bug Fixes & Optimizations

Fixed Chrome Zoom-Out Bug

  • Previously, users on Chrome could continue zooming out after zooming in, causing unwanted white space.
  • Implemented a gesture tracking system to detect when users shrink below 100% scale and automatically reset them to full size.

Prevented Page Zooming While Keeping Image Zoom Functional

  • Stopped touch-action: none; from globally blocking gestures while still preventing page zoom.
  • Adjusted event listeners to allow pinch gestures only on enlarged images, keeping the rest of the site zoom-restricted.

Refined Event Listener Handling

  • Ensured gesturestart, gesturechange, and gestureend properly manage zoom interactions.
  • Moved all zoom-related event handling into a single useEffect hook, reducing unnecessary event listeners.
  • Fixed an issue where zoom restrictions weren‚Äôt consistently applying on iOS and Chrome.

BETA Version 1.2 Patch Notes

Release Date: February 25th, 2025

New Features & Improvements

Image Loading & Stability Enhancements

  • Preloaded Enlarged Images: Enlarged images now load instantly instead of reloading every time they are viewed.
  • Persistent Image Order: The randomization of human and AI images is now stored in localStorage, preventing images from switching positions after a refresh.
  • Loading Indicators: Implemented a loader over image containers while images are loading, ensuring better feedback on slower connections.

Bug Fixes & Optimizations

Image Pair Randomization & Flickering Fixes

  • Fixed Position Switching on Refresh: Image pairs now maintain their initial order throughout the puzzle, eliminating flickering and re-randomization on reload.
  • Cached Image Order: The daily puzzle is now saved in localStorage, ensuring that once an image order is set, it remains unchanged until the next puzzle.

Submit Button Activation Fixes

  • Submit Button Only Activates When All Pairs Are Selected: Fixed an issue where selecting only the last image incorrectly enabled the submit button.
  • Deselecting an Image Properly Disables the Submit Button: If a selection is removed, the submit button will now deactivate unless all pairs remain selected.
  • Clearing Selections Resets Button State: Pressing "Clear" correctly disables the submit button and resets the selection state.
